## Rules

### Rule A: Single primary category, multiple facets
Each insight document has exactly one primary category (determines filename). Facets are used for cross-referencing. Facet values must come from the controlled vocabulary above — no ad-hoc terms.

### Rule B: "Who is responsible?" determines primary category
When an insight spans multiple domains, the domain that "owns" the core idea gets the primary category. Other domains are expressed as facets.
Example: "I prefer systems thinking when approaching product decisions" → primary: thinking-patterns, facets: product, decision-making.

### Rule C: Cross-cutting concepts become topics, not categories
If a concept keeps appearing across many primary categories (e.g., "simplicity"), it becomes a topic collection in memory/curated/topics/ that aggregates entries from multiple Layer 3 files via facet queries — not a new Layer 3 category.

Heuristic: If a concept keeps cutting across many primary categories, it likely should not be a primary category, but rather a dimension or topic.
